在 windows 架設 hexo 寫作環境筆記

目標

用 MarkdownPad 2 寫文章，用 Github 當 Blog 空間。

要下載安裝的東西

  • MarkdownPad 2

    若是使用 Windows 10 可能會無法顯示 Markdown 的畫面，參考官方的解法需再安裝 Awesomium 才能正常顯示。

  • node.js

    我用的是 4.4.7LTS。

  • Git for Windows

    要把 Hexo 推上 Github 用的。

安裝 Hexo

安裝好 noode.js 後，在命令提示字元或 git bash 輸入下面兩行，以 npm 安裝 hexo。

npm install hexo-cli -g
npm install hexo --save

建立 Blog

cd 至想要的目錄後輸入

hexo init

如此就會產生 Hexo 的整個資料夾

在建立 Blog 後，進入該目錄輸入 npm install 以安裝相依套件。

cd blog
npm install

接著可輸入

hexo s -p 8080

打開瀏覽器進入 localhost:8080 即可看到 Blog 在你眼前，如果看不到就是你眼睛業障重或者某個步驟有問題。

開始寫文章

cd 至 Blog 的目錄，輸入

hexo new "this is first article"

會在 /source/_posts 產生 this-is-first-article.md，文章標題輸入中文也是可以的，空白會自動變成 - 符號。

接著就可以開始用 MarkdownPad 來寫文章了。

還沒要發佈的草稿

把文章放在 /source/_drafts 目錄下，該文章就不會發佈，要預覽草稿文章只要在啟動 Server 的時候帶個 --draft 參數即可，大概像這個樣子：

hexo s -p 8080 --draft

在新增文章的時候也可以直接新增至草稿區

hexo new draft "this is title"

把草稿變為正式文章(檔案名稱雖然為 this-is-title，但把 - 換成空格也是可以的)

hexo publish "this is title"

其實手動把檔案從 /source/_drafts 移到 /source/_posts 也可以，感覺上好像還比較簡單。

在文章中插入圖片

這部分有點麻煩，在 /source 建立一目錄，例如 /source/img，然後把圖片丟進去，假設圖片是 gogog.jpg，在文章中輸入 ![](/gogogo.jpg) 就會在 hexo 中正常顯示。

BUT!!

這樣的話在 MarkdownPad 中無法顯示，一個簡單的方法就是先把整個圖片資料夾複製到 /source/_posts 中，文章寫好後再刪掉。猜測應該可以透過修改生成 Blog 的 code 來實現更方便的方法。

更換佈景主題

將佈景主題放在 /themes，然後在 /config.yml 中將 theme 欄位改成你的佈景主題目錄名稱即可。

至於佈景主題該去哪下載呢?

Hexo 官方展示了不少主題，找個喜歡的點進去，有些作者會直接附上該主題的 Github 連結，如果沒有的話再自己 Google 該主題名稱通常就可以找到了。

佈署到 Github

首先要先有一個 [username].github.io 的 repo，假設 Gtihub 使用者名稱是 yoyoboy，就建立一個 yoyoboy.github.io 的 repo。

打開 /themes/_config.yml 在 #Deployment 的地方設定佈署的資訊。

#Deployment
deploy:
    type: git
    repo: https://github.com/yoyoboy/yoyoboy.github.io.git
    branch: master

接著輸入

hexo d -g

成功的話，就會看到

INFO  Deploy done: git

並可以在 https://yoyoboy.github.io 瀏覽你的 Blog。

佈署失敗的錯誤訊息

  • 沒設定 git 使用者

    Please tell me who you are.

    用下面的指令設定一下。

    git config --global user.email "you@example.com"
git config --global user.name "Your Name"

  • 沒安裝 hexo-deployer-git

    ERROR Deployer not found: git

    可以在 Blog 目錄下輸入

    npm install

    或者

    npm install hexo-deployer-git --save

    進行安裝

小技巧

  • hexo clean

    這個指令可以清除 Hexo 所 generate 的東西。在某些主題裡會有 tag 的列表，假設之前不小心在文章寫錯了 tag，已經被加入列表中又不知道怎麼刪掉，可以靠這個指令達成。

結論

其實在 hexo init 之後，自動產生的那篇 Hello World 文章裡面就都有教了 XDD
